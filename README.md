## Vamos praticar 🚀?! 

### 👽 Te desafio a...

1. #### Criar sua conta no Github 😺:
    - Essencial para alcançarmos nossos objetivos.
- [ **X** ] [Meu GitHub](https://github.com/costa-lucassouza)
              
2. #### Criar um Repositório 📄:

    - Crie um novo repositório no GitHub, nomeando-o como **"desafio-github-markdown"** ou outro nome.
    - Adicione o seu README.md ao criar o repositório.
    - Utilize o Github.dev para editar seu projeto diretamente com o editor web do Github.
    - Lembre-se: o repositório deve ser público.

- [ **X** ] [Meu Repositorio](https://github.com/costa-lucassouza/DESAFIO-GITHUB-MARKDOWN)


3. #### Colaboração 🤝:

    - Adicione um colaborador ao seu repositório (pode ser um colega de curso ou você mesmo com uma conta diferente).
    - Ambos devem fazer um clone do repositório em suas máquinas locais.
    - Realizem alterações, Commits, abra issues e pull requests.

4. #### Formatação com Markdown:

    - Este projeto tem como objetivo aplicar, na prática, os conhecimentos adquiridos durante os estudos de Git, GitHub e formatação com Markdown. A proposta foi originalmente elaborada pela professora [Aline Antunes](https://github.com/alinealien), mas este repositório e seu conteúdo — incluindo a estrutura e o arquivo README.md — foram totalmente modificados e adaptados por mim para refletir meu estilo de organização, aprendizado e apresentação.

    - Todo o processo foi documentado de forma clara utilizando a linguagem Markdown.

    - Este desafio foi uma oportunidade criativa de transformar um exercício técnico em uma experiência personalizada de aprendizado.
